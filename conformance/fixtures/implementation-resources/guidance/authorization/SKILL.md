# Authorization review

Consult the resolved SeedSpec, end-user decisions, and actual project before
recommending an authorization approach. Identify actors, protected resources,
authority boundaries, and failure behavior. Do not add accounts, roles, or
permissions that the solution intent does not require.

This package-scoped skill is implementation guidance. It does not add behavior
to the SeedSpec or replace its acceptance criteria.
