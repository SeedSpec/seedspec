# Scheduled service implementation profile

Use a separately operated service only when it is justified by fidelity,
formatting, observability, delivery, or integration requirements. The profile
does not prescribe a language, framework, hosting provider, or deployment
workflow.

Confirm an operational owner, approved runtime, secret store, monitoring path,
and incident response expectation before implementation. Use least-privilege
HubSpot and Slack access, make the daily operation idempotent, and record durable
resource and deployment identifiers.

The implementing agent should choose architecture with the end user after
inspecting the target environment and technical preferences.
