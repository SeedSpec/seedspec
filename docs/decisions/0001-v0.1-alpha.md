# ADR 0001: Agent-guided v0.1 alpha

Status: accepted for the private alpha

## Context

SeedSpec exists to give implementing agents reusable product intent, integration context, and testing information. It is not a conventional package manager and cannot enforce the architecture or terminology of the application an agent builds.

The alpha needs enough structure to author several SeedSpec packages, validate them, combine their intent, signal when an integration deserves deeper review, and prove the handoff with a working Allowance Tracker use case.

## Decision

- Use Node.js 20, YAML, Markdown, and JSON Schema for the reference tooling.
- Use `protocol_version: "0.1"` and pre-release package versions while all design remains private and reconsiderable.
- Let providers identify exact capability contract revisions.
- Let consumers record `tested_against` revisions as testing evidence rather than minimum or maximum dependency constraints.
- Resolve a different capability revision with `integration_status: review`; do not reject it merely because versions differ.
- Generate an agent guide and persistent implementation-note and verification-report files in resolved projects.
- Treat application code, local terminology, user data, and actual tests as authoritative evidence during integration.
- Validate the workflow against independently maintained reference applications without making them part of the protocol repository.

## Consequences

The format remains testable and versioned without claiming authority over the final application. Agents receive explicit reasons to plan more carefully while retaining freedom to adapt incoming behavior. Public ecosystem machinery, migrations, registries, signing, and universal generation remain outside the alpha.
