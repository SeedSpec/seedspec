# HubSpot-native implementation profile

Use HubSpot-native reporting, calculated properties, workflows, scheduled
automation, and supported Slack delivery where they can satisfy the core intent.

Inspect the customer's active HubSpot edition and enabled capabilities before
planning. Do not assume a feature exists merely because it appears in general
HubSpot documentation. Prefer durable HubSpot resource identifiers over display
names in implementation notes and verify the previous-day time window at the
customer's configured timezone boundary.

If native behavior cannot calculate, format, schedule, or deliver the metric
faithfully, show the limitation to the user and ask whether to prefer another
profile. Do not silently approximate the intended result.
